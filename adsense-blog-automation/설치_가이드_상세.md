# 📘 처음부터 끝까지 따라하는 설치 가이드 (초보자용 · 최신판)

이 도구는 매일 아침 7시에 **한국어 글 10개 + 영어 글 10개 = 하루 20개**를 자동으로 만듭니다.
- **한국어**: 한국 기준. 네이버 실제 검색량으로 트렌드 5개 + 꾸준한 주제(에버그린) 5개
- **영어**: 미국 기준. 구글 트렌드로 트렌드 5개 + 꾸준한 주제 5개
- 아이폰 앱으로 확인하고, 버튼 한 번으로 워드프레스에 올릴 수 있습니다.

> 컴퓨터는 안 켜도 됩니다. 글 만들기는 인터넷(클라우드)에서 자동으로 돌아갑니다.
> 아래는 **딱 한 번만** 하는 설정입니다. 천천히 따라오세요. 예상 40분~1시간.

준비물 4가지 (아래에서 하나씩 발급): ① 구글 계정 ② Gemini 키 ③ 네이버 검색광고 키 ④ GitHub 계정.
(워드프레스 자동 게시는 나중에 추가해도 됩니다.)

---

# PART 1. Gemini 키 받기 (글 쓰는 엔진 · 무료)

> ⚠️ 제미나이 '앱 구독'과 '키'는 다릅니다. 아래에서 무료 키를 따로 받습니다. 카드 필요 없음.

1. 컴퓨터 인터넷 창에서 **aistudio.google.com** 접속 → 구글 계정으로 로그인.
2. 왼쪽(또는 오른쪽 위) **Get API key / API 키 가져오기** 클릭.
3. **Create API key / API 키 만들기** 클릭 → `AIza...` 로 시작하는 긴 글자가 생깁니다.
4. **복사** 버튼을 눌러 복사하고, 메모장에 잠깐 붙여둡니다. (나중에 씁니다)

---

# PART 2. 네이버 검색광고 키 받기 (한국 검색량 · 무료)

한국어 키워드의 '실제 월간 검색수'를 가져오는 데 씁니다. 개인도 무료 가입됩니다.

1. **searchad.naver.com** 접속 → **신규 가입**(네이버 계정으로) → 유형은 개인 선택 가능.
2. 로그인 후 오른쪽 위 **도구 → API 사용 관리** 클릭.
3. **API 라이선스 등록/발급** 을 누르면 값 3개가 나옵니다. 메모장에 복사:
   - **액세스 라이선스** (긴 글자)
   - **비밀키** (긴 글자)
   - **고객 ID** (숫자. 화면 오른쪽 위 '내 계정'에 뜨는 숫자, 'CUSTOMER_ID')
4. 이 3개는 PART 5에서 사용합니다.

> 지금 네이버 발급이 번거로우면 건너뛰어도 됩니다. 그 경우 한국어도 '구글 트렌드'로
> 대략 순위만 매기게 되고, 정확한 검색수 숫자는 안 나옵니다. 나중에 추가 가능합니다.

---

# PART 3. GitHub 가입 & 저장소 만들기

'저장소'는 프로그램 파일을 올려두는 인터넷 폴더입니다.

1. **github.com** → 오른쪽 위 **Sign up** → 구글 이메일로 무료 가입.
2. 로그인 후 오른쪽 위 **＋ → New repository** 클릭.
3. 입력:
   - **Repository name**: `blog-auto` (영문/숫자, 원하는 이름)
   - **Public** 선택 (무료 기능을 쓰기 위함. 비밀 키는 파일에 안 들어가니 안전)
4. 아래 **Create repository** 클릭.

---

# PART 4. 파일 올리기

### 4-1. 일반 파일 올리기
1. 방금 만든 저장소 화면에서 **Add file → Upload files** 클릭.
2. 컴퓨터의 `adsense-blog-automation` 폴더를 열고, **안에 있는 모든 파일과 폴더**를
   드래그해서 화면에 놓습니다.
3. 초록색 **Commit changes** 클릭. (업로드 완료)

### 4-2. 자동실행 파일은 웹에서 직접 만들기 (중요)
`.github` 폴더는 컴퓨터에서 숨겨져 있어 잘 안 올라갑니다. 그래서 웹에서 직접 만듭니다.

1. 저장소 화면 **Add file → Create new file**.
2. 파일 이름 칸에 **정확히** 입력(슬래시 포함):
   ```
   .github/workflows/daily-blog.yml
   ```
3. 아래 큰 칸에, 채팅에서 드린 **워크플로 내용(daily-blog.yml)** 을 그대로 붙여넣기.
   (내 컴퓨터의 `.github/workflows/daily-blog.yml` 파일을 메모장으로 열어 복사해도 됩니다.)
4. 초록색 **Commit changes** 클릭.

---

# PART 5. 비밀 키 등록 (시크릿)

키를 코드에 직접 안 넣고 안전하게 보관하는 곳입니다.

1. 저장소 위쪽 **Settings** 클릭.
2. 왼쪽 메뉴 **Secrets and variables → Actions**.
3. **New repository secret** 을 눌러 아래를 **하나씩** 추가 (Name=이름, Secret=값).

### 꼭 필요한 것
| Name | Secret(값) |
|---|---|
| `LLM_PROVIDER` | `gemini` |
| `LLM_API_KEY` | PART 1의 Gemini 키 (`AIza...`) |
| `LLM_MODEL` | `gemini-2.5-flash` |
| `LANGS` | `ko,en` |

### 한국어 검색량(네이버) — PART 2를 했다면 추가
| Name | Secret(값) |
|---|---|
| `NAVER_API_KEY` | 네이버 액세스 라이선스 |
| `NAVER_SECRET_KEY` | 네이버 비밀키 |
| `NAVER_CUSTOMER_ID` | 네이버 고객 ID(숫자) |

> 영어(미국)는 따로 키가 필요 없습니다. 기본으로 구글 트렌드를 씁니다.
> 네이버를 건너뛰었다면 `KO_METRICS_PROVIDER` = `trends` 를 추가하세요.

### 워드프레스 자동 게시 — 원할 때만 (나중에 가능)
| Name | Secret(값) |
|---|---|
| `WP_ENABLED` | `true` |
| `WP_SITE` | `https://내블로그.com` |
| `WP_USER` | 워드프레스 아이디 |
| `WP_APP_PASSWORD` | 응용 프로그램 비밀번호(PART 9-보충 참고) |
| `WP_STATUS` | `draft` (검토 후 발행) |

---

# PART 6. 한번 실행해서 테스트

1. 저장소 위쪽 **Actions** 탭 클릭.
2. 안내 버튼이 보이면 **I understand my workflows, enable them** 클릭.
3. 왼쪽에서 **매일 블로그 자동 생성** 클릭 → 오른쪽 **Run workflow → Run workflow**.
4. 잠시 후 목록에 줄이 생깁니다. 클릭해서 진행을 볼 수 있어요.
   - 약 3~10분 뒤 **초록 체크 ✓** = 성공! (글 20개 생성)
   - **빨간 ✗** 이면 줄을 눌러 로그 확인 → 보통 시크릿 이름 오타/누락입니다. 고치고 다시 Run.
5. 성공하면 이후 **매일 한국시간 오전 7시**에 저절로 실행됩니다.

---

# PART 7. 결과 화면을 인터넷에 공개 (GitHub Pages)

1. **Settings → 왼쪽 Pages**.
2. **Source** 를 **Deploy from a branch** 로, **Branch** 를 **main / (root)** 로 선택 → **Save**.
3. 1~2분 뒤 위쪽에 주소가 나옵니다. 뒤에 `/dashboard/` 를 붙인 것이 우리 화면입니다:
   ```
   https://<내아이디>.github.io/blog-auto/dashboard/
   ```
4. 이 주소를 컴퓨터에서 먼저 열어 글 20개가 보이는지 확인.
   - **데이터 주소**도 메모해 두세요(아이폰 설정에 씁니다):
     `https://<내아이디>.github.io/blog-auto/dashboard/data/latest.json`

---

# PART 8. 아이폰에 앱으로 설치

1. 아이폰 **사파리(Safari)** 로 PART 7의 `.../dashboard/` 주소 접속. (크롬 아님)
2. 화면 아래 가운데 **공유 버튼**(네모에 위 화살표) 탭.
3. 목록을 내려 **홈 화면에 추가** → 오른쪽 위 **추가**.
4. 홈 화면에 생긴 🚀 아이콘을 누르면 전체화면 앱처럼 열립니다.

---

# PART 9. 앱에서 설정

앱 오른쪽 위 **⚙️(톱니바퀴)** 를 누릅니다.

1. **데이터 주소** 칸에 PART 7의 `.../data/latest.json` 주소 붙여넣기.
   → 매일 7시 새 글이 자동으로 앱에 들어옵니다.
2. (폰에서 바로 게시하려면) **워드프레스** 사이트·아이디·앱비밀번호 입력, 게시 방식은 처음엔 **임시저장**.
3. **저장**.

이제 글 카드에서:
- **📤 게시** — 폰에서 바로 워드프레스로 올리기
- **📋 HTML / 텍스트** — 복사해서 붙여넣기
- 위쪽 필터로 **한국어/EN**, **높은/낮은 경쟁력** 나눠 보기
- 각 카드에 **월검색량·경쟁도·꾸준함%** 표시

### 9-보충. 워드프레스 응용 프로그램 비밀번호 만드는 법
워드프레스 관리자 → **사용자 → 프로필** → 맨 아래 **응용 프로그램 비밀번호** →
이름(예: auto-blog) 입력 → 추가 → 생성된 `abcd efgh ...`(공백 포함) 전체 복사.
(자동 게시면 PART 5의 `WP_APP_PASSWORD`에, 폰 게시면 PART 9 앱 설정에 입력.)

# 🔧 자주 막히는 곳

| 증상 | 해결 |
|---|---|
| Actions 빨간 ✗ | 줄 클릭 → 로그 확인. 보통 `LLM_API_KEY` 오타/누락. 시크릿 이름 정확히 |
| 한국어 검색량이 '관심도'로만 나옴 | 네이버 시크릿 3개가 정확한지, `KO_METRICS_PROVIDER`가 있다면 `naver`인지 확인 |
| Pages 주소 404 | 저장 후 2분 기다리기. Branch가 `main`/`root`인지 확인 |
| 앱에 글이 안 보임 | ⚙️ 데이터 주소가 `.../data/latest.json` 까지 정확한지, ↻ 새로고침 |
| 폰에서 게시 실패 | CORS 문제. README 9-4의 워드프레스 코드 추가, 또는 자동 게시(PART 5)로 전환 |
