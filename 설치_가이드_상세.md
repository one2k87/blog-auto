# 📘 처음부터 끝까지 따라하는 설치 가이드 (초보자용 · 최신판)

이 도구는 매일 아침 7시에 **3개 카테고리**(금융/재테크 · 건강/생활 · 경제/IT)에서
각각 **6슬롯**(저경쟁 시리즈2+단일2, 시즌 시리즈1+단일1)씩 자동으로 글을 만듭니다.
- 글마다 **이미지 1개 자동 생성**, 문단 사이 **광고 자리 + 유도문장**, 요약표, 내부링크 포함
- 저경쟁/시즌 주제를 매번 **자동 판별**해서 배치
- 컴퓨터 없이 **아이폰 앱에서 확인·게시·"지금 실행"까지** 제어

> 아래는 **딱 한 번만** 하는 설정입니다. 순서대로 천천히 따라오세요. 예상 40분~1시간.

준비물: ① 구글 계정 ② Gemini 키 ③ 네이버 검색광고 키 ④ GitHub 계정.
(워드프레스 자동 게시는 나중에 추가 가능)

---

# PART 1. Gemini 키 받기 (글·이미지 엔진 · 무료)

> ⚠️ 제미나이 '앱 구독'과 '키'는 다릅니다. 아래에서 무료 키를 따로 받습니다. 카드 불필요.

1. 컴퓨터 브라우저에서 **aistudio.google.com** 접속 → 구글 로그인.
2. **Get API key / API 키 가져오기 → Create API key** 클릭.
3. `AIza...` 로 시작하는 키를 복사해 메모장에 보관.

> 이미지도 이 Gemini 키로 자동 생성합니다(Imagen). 만약 이미지 생성이 안 되면 나중에
> 이미지를 끄거나(OpenAI로 교체) 자리 표시로 대체되니, 일단 진행하세요.

---

# PART 2. 네이버 검색광고 키 받기 (한국 검색량 · 무료)

저경쟁 주제를 '실제 검색량'으로 골라내는 데 씁니다. 개인도 무료 가입됩니다.

1. **searchad.naver.com** 가입 → 로그인.
2. 오른쪽 위 **도구 → API 사용 관리 → API 라이선스 발급**.
3. 값 3개를 메모장에 복사:
   - **액세스 라이선스**
   - **비밀키**
   - **고객 ID**(숫자)

> 네이버 발급이 번거로우면 건너뛰어도 됩니다(주제를 AI 판단으로만 고름). 나중에 추가 가능.

---

# PART 3. GitHub 가입 & 저장소 만들기

1. **github.com → Sign up** (구글 이메일로 무료 가입).
2. 오른쪽 위 **＋ → New repository**.
3. **Repository name**: `blog-auto` → **Public** 선택 → **Create repository**.

---

# PART 4. 파일 올리기

### 4-1. 일반 파일
1. 저장소 화면 **Add file → Upload files**.
2. 컴퓨터의 `adsense-blog-automation` 폴더 **안의 모든 파일·폴더**를 드래그 → **Commit changes**.

### 4-2. 자동실행 파일은 웹에서 직접 만들기 (중요)
`.github` 폴더는 숨김이라 잘 안 올라갑니다.
1. **Add file → Create new file**.
2. 파일 이름 칸에 정확히: `.github/workflows/daily-blog.yml`
3. 내 컴퓨터의 같은 파일 내용을 복사해 붙여넣기 → **Commit changes**.

---

# PART 5. 비밀 키 등록 (시크릿)

저장소 **Settings → Secrets and variables → Actions → New repository secret** 에서 하나씩 추가.

### 필수
| Name | 값 |
|---|---|
| `LLM_PROVIDER` | `gemini` |
| `LLM_API_KEY` | PART 1 Gemini 키 |
| `LLM_MODEL` | `gemini-2.5-flash` |
| `IMAGE_PROVIDER` | `gemini` (이미지도 Gemini로) |

### 한국 검색량(네이버) — PART 2 했다면
| Name | 값 |
|---|---|
| `METRICS_PROVIDER` | `naver` |
| `NAVER_API_KEY` | 액세스 라이선스 |
| `NAVER_SECRET_KEY` | 비밀키 |
| `NAVER_CUSTOMER_ID` | 고객 ID(숫자) |

### 카테고리 바꾸고 싶을 때만 (선택)
기본값이 금융/건강/경제IT 3개입니다. 바꾸려면 `CATEGORIES_JSON` 에 아래처럼 넣으세요:
```
[{"name":"금융/재테크","desc":"대출,카드,정부지원금,연금,세금"},{"name":"건강/생활","desc":"다이어트,영양제,탈모,수면"},{"name":"경제/IT","desc":"물가,금리,AI도구,앱 사용법"}]
```

### 워드프레스 자동 게시 — 원할 때만 (나중에 가능)
`WP_ENABLED`=`true`, `WP_SITE`, `WP_USER`, `WP_APP_PASSWORD`, `WP_STATUS`=`draft`, `BLOG_URL`=사이트주소.

# PART 6. 한번 실행해서 테스트

1. 저장소 위쪽 **Actions** → 안내 나오면 **활성화** 버튼 클릭.
2. 왼쪽 **매일 블로그 자동 생성** → 오른쪽 **Run workflow → Run workflow**.
3. 5~15분 뒤 **초록 체크 ✓** = 성공(3개 카테고리 × 6슬롯 생성).
   - 빨간 ✗ 면 줄을 눌러 로그 확인 → 보통 시크릿 오타/누락.
4. 이후 **매일 오전 7시** 자동 실행.

---

# PART 7. 결과 화면 인터넷 공개 (GitHub Pages)

1. **Settings → Pages**.
2. **Source: Deploy from a branch → main / (root) → Save**.
3. 1~2분 뒤 주소 확인. `/dashboard/` 를 붙인 것이 우리 화면:
   `https://<내아이디>.github.io/blog-auto/dashboard/`
4. 데이터 주소도 메모: `https://<내아이디>.github.io/blog-auto/dashboard/data/latest.json`

---

# PART 8. 아이폰 실행 제어용 토큰 만들기 (폰에서 '지금 실행')

폰에서 버튼으로 생성을 돌리려면 GitHub 토큰이 하나 필요합니다.

1. github.com → 오른쪽 위 프로필 → **Settings → Developer settings**
   → **Personal access tokens → Fine-grained tokens → Generate new token**.
2. 설정:
   - **Repository access**: Only select repositories → `blog-auto` 선택
   - **Permissions → Repository permissions → Actions: Read and write**
3. **Generate token** → `github_pat_...` 복사(한 번만 보임).

---

# PART 9. 아이폰에 앱 설치

1. 아이폰 **사파리(Safari)** 로 PART 7의 `.../dashboard/` 주소 접속.
2. 아래 **공유 버튼(□↑) → 홈 화면에 추가 → 추가**.
3. 홈 화면 🚀 아이콘으로 열면 전체화면 앱으로 실행.

---

# PART 10. 앱에서 설정 (폰이 통제 센터)

앱 오른쪽 위 **⚙️**:

1. **데이터 주소**: PART 7의 `.../data/latest.json` 붙여넣기.
2. **폰에서 실행(GitHub)**:
   - 사용자명(owner) = 내 GitHub 아이디
   - 저장소(repo) = `blog-auto`
   - 액세스 토큰 = PART 8의 `github_pat_...`
3. (게시하려면) **워드프레스** 정보 입력.
4. **저장**.

이제 앱에서:
- 상단 **▶ 버튼** → "지금 생성 실행"(폰에서 바로 글 생성 시작)
- **↻** → 새 결과 불러오기
- 카테고리/저경쟁·시즌 **필터**로 나눠 보기
- 글마다 **📤 게시 / 📋 복사 / 미리보기**

# 🔧 자주 막히는 곳

| 증상 | 해결 |
|---|---|
| Actions 빨간 ✗ | 로그 확인. `LLM_API_KEY` 오타/누락이 대부분 |
| 이미지가 '자리 표시'로만 나옴 | Gemini 키에 이미지(Imagen) 권한이 없을 수 있음. `IMAGE_PROVIDER`를 `none`으로 두거나 OpenAI로 교체 |
| 저경쟁 검색량이 안 보임 | 네이버 시크릿 3개 확인, `METRICS_PROVIDER=naver` |
| 앱 ▶ 실행 실패 | 토큰 권한(Actions Read/Write)·owner/repo 철자 확인 |
| Pages 404 | 저장 후 2분 대기, Branch main/root 확인 |
| 폰 게시 실패 | CORS. README 9-4 워드프레스 코드 추가 또는 자동 게시로 전환 |

# 💰 비용 / ⚠️ 주의

- **글 생성(Gemini 텍스트)**: 무료 한도(하루 1,500회) 안. 3카테고리라도 넉넉.
- **이미지 생성**: 제공자에 따라 과금될 수 있음. 무료로 쓰려면 `IMAGE_PROVIDER=none`(자리 표시).
- **AdSense**: 자동 글 무검수 대량 발행은 위험. 처음엔 `draft`로 두고 **검토 후 발행**, 숫자·링크는 한 번 확인.
- 저경쟁·시리즈·카테고리 내부링크가 신생 블로그의 상위노출·체류시간·반복수익에 유리합니다.
